Doc Settings

Name
Type
Description
type
'markdown' | 'page'

The default type is markdown. If you set it to page, the import option must be set too. When type is page, the page will be rendered by the module you specified with the import option. If the type is not set and import is set, the type will be set to page automatically.

import
string

Specify the module id of a tsx or jsx file. The file will be used to render the doc page as long as the type is not set to markdown. The module id must be a relative path.

title
string

Specify the document title. If not set, the first heading founded in this doc will be used.

summary
string

Specify the summary of the doc. If not set, will be set to the first paragraph of the doc. The summary will be searched.

tags
string[]

Set the tags of the current doc. The tags will be searched.

slug
string

The slug of the doc. The value will be used to build the doc url.

language
string

The document language. If not set, will be resolved by the doc path.

lang
string

Alias for language.

translationKey
string

If two or more docs has the same translationKey, means these docs are just translation of each other. Most time you don't need to set this value by yourself, if you use the filename suffix to specify the language of the doc. In this case, the translation key will be resolved by the doc path automatically.

hidden
boolean

If you set hidden to true, this doc will be hidden from the menu in sidebar. Note that this doc is still accessible if someone knows the url of this doc.

fullscreen
boolean

Take over the fullscreen when render this doc. It's useful when the type is page.

menu
string

The name of menu you want to put this doc into.

pageTitle
string

Set the browser tab title. If not set, title will be used.

docPageTitleFormat
string

Set the format of pageTitle. Override the docPageTitleFormat config in the ui settings.

*
any

Other extra settings defined by the user.
